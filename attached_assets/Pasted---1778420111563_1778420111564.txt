Создай веб-приложение для адаптации резюме под вакансию. 
Приложение состоит из двух частей: бэкенд на Python (FastAPI) 
и фронтенд на React.

═══════════════════════════════════════
АРХИТЕКТУРА
═══════════════════════════════════════

Бэкенд (Python + FastAPI):
- Файл backend/main.py
- Два эндпоинта:
  POST /analyze — принимает текст вакансии и резюме, возвращает JSON 
  с требованиями, критичностью и статусом соответствия.
  POST /adapt — принимает текст вакансии, резюме и ответы пользователя, 
  возвращает адаптированное резюме с тегами изменений.
- Настрой CORS чтобы фронтенд мог обращаться к бэкенду.
- Промпты для LLM вынеси в отдельный файл backend/prompts.py чтобы 
  их можно было легко найти и заменить.

Фронтенд (React + Vite):
- Папка frontend/
- Все запросы к LLM идут через бэкенд, не напрямую из браузера.

═══════════════════════════════════════
ЭКРАН 1 — ЗАГРУЗКА
═══════════════════════════════════════

Два варианта ввода резюме:
- текстовое поле (вставить текст);
- загрузка файла (PDF или DOCX) с извлечением текста.

Текстовое поле для вакансии.

Кнопка «Анализировать» — отправляет запрос на POST /analyze.
Во время загрузки показывай spinner с текстом «Анализирую соответствие...».

═══════════════════════════════════════
ЭКРАН 2 — АНАЛИЗ СООТВЕТСТВИЯ
═══════════════════════════════════════

Вверху — Match Score в процентах. Считается на фронтенде по формуле:
Score = Σ(вес × оценка) / Σ(весов) × 100%

Веса: обязательное (must) = 2, желательное (nice) = 1.
Оценки: confirmed = 1, partial = 0.5, missing = 0.

Match Score пересчитывается в реальном времени по мере того, как 
пользователь заполняет ответы на красные карточки.

Список карточек требований, отсортированный по статусу:
1. Красные карточки (status: missing) — вверху. Показывают требование 
   и текстовое поле для ответа пользователя с placeholder 
   «Есть ли у вас такой опыт? Опишите кратко...».
2. Жёлтые карточки (status: partial) — в середине. Информационные, 
   без действий. Подпись: «Будет усилено при адаптации резюме».
3. Зелёные карточки (status: confirmed) — внизу. Информационные, 
   без действий.

Каждая карточка показывает:
- текст требования;
- бейдж критичности: «обязательное» (синий) или «желательное» (серый);
- бейдж статуса с цветом карточки.

Кнопка «Адаптировать резюме» — отправляет запрос на POST /adapt,
передавая текст вакансии, резюме и ответы пользователя на красные карточки.
Во время загрузки показывай spinner с текстом «Адаптирую резюме...».

═══════════════════════════════════════
ЭКРАН 3 — АДАПТИРОВАННОЕ РЕЗЮМЕ
═══════════════════════════════════════

Показывает адаптированное резюме с подсветкой изменений.
Все фрагменты внутри тегов <change>...</change> подсвечиваются 
жёлтым фоном (#FEF3C7) с нижней границей (#D97706).

Показывает обновлённый Match Score с приростом: «71% → 86%».

Три кнопки скачивания:
- «Скачать TXT» — plain text без тегов;
- «Скачать DOCX» — Word-документ без тегов;
- «Скачать PDF» — через браузерную печать с CSS.

Кнопка «Начать заново» — возвращает на экран 1.

ПРОМПТЫ ДЛЯ LLM (файл backend/prompts.py)
═══════════════════════════════════════

ПРОМПТ 1 (для эндпоинта /analyze):

ANALYZE_PROMPT = """
Ты — эксперт по анализу вакансий и резюме. Твоя задача — извлечь 
требования из вакансии, оценить их критичность и сопоставить с резюме 
кандидата.

Выполняй задачу последовательно в три шага. Пользователь видит только 
финальный результат — промежуточные рассуждения не нужны.

ШАГ 1. ИЗВЛЕЧЕНИЕ ТРЕБОВАНИЙ

ИСТОЧНИКИ
Извлекай только из блоков, которые описывают ожидания от кандидата. 
Типичные заголовки: «Требования», «Обязательные требования», «Что важно», 
«Ожидания», «Must have», «Будет плюсом», «Nice to have», «Дополнительно» 
и аналогичные по смыслу.

Не извлекай из блоков: «О компании», «Задачи», «Обязанности», 
«Что мы предлагаем», «Условия» и аналогичных.

КРИТЕРИИ ОТБОРА
Включай требование, если оно одновременно:
1. Описывает навык, знание, опыт или квалификацию кандидата.
2. Поддаётся проверке по резюме.
3. Не входит в список исключений.

Перед включением задай себе вопрос: «Смогу ли я по резюме однозначно 
ответить, соответствует ли кандидат этому требованию?» Если нет — 
не включай.

СПИСОК ИСКЛЮЧЕНИЙ — всегда отбрасывай:
- Личностные качества: ответственность, стрессоустойчивость и аналогичные.
- Абстрактные установки: желание развиваться, активная жизненная позиция 
  и аналогичные.
- Общие soft-формулировки без конкретики.
- Требования к личным обстоятельствам: готовность к командировкам и т.д.

ПРАВИЛА ФОРМУЛИРОВКИ
- Каждое требование — одно конкретное умение, знание или опыт.
- Независимые навыки разбивай на отдельные требования.
- Требование с уточнением оставляй как одно: «опыт от 3 лет, идеально 
  в HR-Tech» — одно требование.
- Формулируй нейтрально, без оценочных слов.

ШАГ 2. КЛАССИФИКАЦИЯ КРИТИЧНОСТИ

must — без этого кандидат отсеивается на скрининге.
Признаки: «требуется», «обязательно», «не менее N лет», блок «Требования».

nice — усиливает кандидата, но не блокирует.
Признаки: «будет плюсом», «желательно», блок «Будет плюсом».

При неоднозначности — ставь must.
Классифицируй по тексту вакансии, а не по своим представлениям.

ШАГ 3. СОПОСТАВЛЕНИЕ С РЕЗЮМЕ

confirmed — в резюме есть явное упоминание, прямо соответствующее 
требованию.

partial — в резюме есть релевантный факт, но сформулирован иначе. 
При адаптации резюме это соответствие будет усилено автоматически.

missing — в резюме нет ни явного, ни косвенного упоминания.

Правила:
- Не интерпретируй расширительно.
- Для partial соответствие должно быть честным. Если сомневаешься — 
  ставь missing.

ФОРМАТ ВЫВОДА
Верни только JSON без пояснений и markdown:
{
  "requirements": [
    {
      "id": 1,
      "text": "текст требования",
      "criticality": "must",
      "status": "confirmed"
    }
  ]
}

Текст вакансии:
{{vacancy_text}}

Резюме кандидата:
{{resume_text}}
"""

ПРОМПТ 2 (для эндпоинта /adapt):

ADAPT_PROMPT = """
Ты профессиональный карьерный консультант и эксперт в области написания 
резюме. Твоя задача — адаптировать резюме кандидата под вакансию, не 
выдумывая фактов и не добавляя несуществующий опыт.

ШАГ 1. АНАЛИЗ ВАКАНСИИ
Выдели ключевые обязанности, навыки, опыт и отраслевую лексику.

ШАГ 2. АНАЛИЗ РЕЗЮМЕ
Определи что уже отражено, что подразумевается, что добавить из ответов.

ШАГ 3. ОБНОВЛЕНИЕ РЕЗЮМЕ

РАЗДЕЛ SUMMARY / О СЕБЕ
- Пиши связным текстом без отдельного заголовка перед текстом.
- Если раздел есть — перепиши под вакансию.
- Если раздела нет — создай и добавь в начало резюме.
- Оберни в тег <change>...</change>.

РАЗДЕЛ ОПЫТ РАБОТЫ
- Перепиши каждый bullet-пункт под задачи вакансии.
- Переупорядочь пункты — наиболее релевантные ставь первыми.
- Добавь задачи, если они реально выполнялись, но не отражены в резюме.
- Используй action verbs, цифры — только если они были в оригинале.
- Используй терминологию вакансии там, где она честно соответствует опыту.
- Переписанные и добавленные пункты оберни в тег <change>...</change>.
- Неизменённые пункты оставь без тегов.

ИНТЕГРАЦИЯ ОТВЕТОВ КАНДИДАТА
- Инструменты и технологии → добавь в раздел «Навыки», оберни в <change>.
- Опыт и задачи → добавь в релевантное место работы, оберни в <change>.
- Используй только факты из ответов — не додумывай.
- Пустой ответ — ничего не добавляй.

ОСТАЛЬНЫЕ РАЗДЕЛЫ
- Сохраняй без изменений: образование, курсы, достижения.
- Не удаляй информацию из резюме.
- ЗАПРЕЩЕНО трогать: «Цель», «Образование», «Курсы», «Достижения».

ВАЖНО
- Не придумывай проекты, навыки или опыт.
- Не меняй факты, даты, названия компаний.
- Не меняй стиль разделов, которые не затронуты адаптацией.

ФОРМАТ ВЫВОДА
Верни только JSON без пояснений и markdown:
{
  "resume_updated": "полный текст резюме с тегами <change>"
}

Текст вакансии:
{{vacancy_text}}

Резюме кандидата:
{{resume_text}}

Ответы кандидата:
{{user_answers}}
"""
